# Relational Database Design and Implementation

## Project Overview
This project demonstrates the design and implementation of a relational database system using SQL. The objective was to design a database based on a real world scenario, develop an Entity Relationship (ER) model, and implement the database structure using SQL.

## Problem Description
The task was to design a relational database system for a chosen case study. The database needed to represent multiple entities and their relationships while following proper database design principles.

The system was required to:

- Design an Entity Relationship (ER) diagram with approximately 10–12 related entities.
- Define primary keys and foreign keys for entity relationships.
- Normalize the database structure to Third Normal Form (3NF).
- Implement the database using SQL Data Definition Language (DDL).
- Populate the database tables using SQL Data Manipulation Language (DML).
- Insert sample data into each table.
- Demonstrate the database functionality through SQL queries.

## Implementation Requirements
The database implementation includes:

- Table creation using SQL (DDL)
- Defining constraints such as Primary Keys and Foreign Keys
- Inserting sample records into each table (DML)
- Testing the database with SQL queries
